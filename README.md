# Задание 1: Работа со стеком (добавление элемента)

## Описание задачи
Имеется указатель P1 на вершину непустого стека. Элементы стека представлены в виде 
связанных узлов-записей типа TNode, где каждый узел содержит целочисленное значение Data 
и указатель Next на следующий элемент. Вершиной стека считается первый элемент цепочки, 
а поле Next последнего элемента равно nullptr. Требуется добавить новый элемент со 
значением D в стек и вывести адрес P2 новой вершины стека. Известно, что стек непустой.

**Входные данные** (ввод с клавиатуры):
- Целое число D — значение для нового элемента
- Последовательность целых чисел для формирования начального стека (признак окончания 
  ввода — ввод символа 'q')

**Выходные данные:**
- Адрес P2 новой вершины стека (после добавления элемента)
- Содержимое стека после добавления элемента

## Структура программы

Логика программы разбита на отдельные методы с четкими ответственностями:

| Класс / Метод                | Ответственность |
|------------------------------|------------------|
| `TNode::TNode(int data)`     | Конструктор узла. Инициализирует значение узла и обнуляет указатель на следующий узел (`next`). |
| `Stack::Stack()`             | Конструктор стека. Создает пустой стек, устанавливая указатель на вершину (`top`) в `nullptr`. |
| `Stack::push(int node_data)` | Добавляет новый узел с данными `node_data` на вершину стека. |
| `Stack::show()`              | Выводит все элементы стека на экран. |
| `Stack::delete_data(Stack&)` | Освобождает память, занятую узлами переданного стека. |
| `Stack::show_address()`      | Отображает адреса памяти всех узлов стека. |
| `Stack::get_top_value()`     | Возвращает значение, хранящееся на вершине стека, не удаляя его. |
| `Stack::is_empty()`          | Проверяет, пуст ли стек (возвращает `true`, если элементов нет). |
| `StackWork::fill_amount_of_digits()` | Определяет количество элементов для заполнения стека. |
| `StackWork::fill_stack()`    | Заполняет стек заданным количеством элементов (вызывает `push`). |
| `StackWork::show_work()`     | Демонстрирует работу стека: заполнение, вывод, очистку и т.д. |

# Задание 2: Работа с очередью (извлечение элементов)

## Описание задачи
Дано число N (> 0) и указатели P1 и P2 на начало и конец непустой очереди. 
Очередь моделируется цепочкой связанных узлов-записей типа TNode. Требуется 
извлечь из очереди N начальных элементов и вывести их значения. Если очередь 
содержит менее N элементов, то извлечь все ее элементы. После извлечения 
элементов необходимо освобождать память, которую они занимали.

**Входные данные** (ввод с клавиатуры):
- Целое число N — количество элементов для извлечения (N > 0)
- Последовательность целых чисел для формирования начальной очереди (признак окончания 
  ввода — ввод символа 'q')

**Выходные данные:**
- Значения извлеченных элементов (в порядке их извлечения из очереди)
- Новые адреса начала и конца очереди (для пустой очереди дважды вывести nullptr)

## Структура программы

Логика программы для работы с очередью (queue) разбита на отдельные методы с четкими ответственностями:

| Класс / Метод | Ответственность |
|:---------------|:-----------------|
| `Node` | Структура узла для очереди. Содержит поле `data` для хранения целочисленных данных и указатель `next` на следующий узел. |
| `PNode` | Упрощает работу с указателями на узлы очереди. |
| `Queues::add_nodes()` | Добавляет новый узел с переданными данными в конец очереди. Работает с указателями на голову (`node_head`) и хвост (`node_tail`). |
| `Queues::show_source()` | Выводит на экран все элементы исходной очереди от головы до хвоста. |
| `Queues::delete_nodes()` | Удаляет заданное количество элементов (`amount_to_delete`) из начала очереди. Модифицирует указатель на голову (`node_head`). |
| `Queues::show_fixed()` | Выводит на экран все элементы измененной очереди после удаления. |
| `Lists::fill_amount_of_digits()` | Определяет количество элементов для заполнения очереди. |
| `Lists::fill_amount_to_delete()` | Определяет количество элементов, которые нужно удалить из очереди. |
| `Lists::fill_digits()` | Заполняет очередь заданным количеством элементов.  |
| `Lists::show_work()` | Основной демонстрационный метод. Координирует весь процесс: создание очереди, наполнение данными, отображение исходной очереди, удаление элементов и показ финального состояния. |

- Содержимое списка (выводится на экран для контроля)
